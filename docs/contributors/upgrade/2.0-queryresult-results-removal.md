# Upgrade note: `QueryResult.results` removal in 2.0

In Tango 1.x, `QuerySet.fetch()` returns a `QueryResult` that is iterable and provides `toArray()`. The `results` property remains available as a compatibility surface and is deprecated.

In Tango 2.0, `QueryResult.results` is removed.

## What to change

### Iterate results directly

Previously:

```ts
const page = await PostModel.objects.query().filter({ published: true }).fetch();
for (const post of page.results) {
    post.title;
}
```

Now:

```ts
const page = await PostModel.objects.query().filter({ published: true }).fetch();
for (const post of page) {
    post.title;
}
```

### Convert to an array when you need array helpers

Previously:

```ts
const page = await PostModel.objects.query().fetch();
page.results.map((row) => row.id);
```

Now:

```ts
const page = await PostModel.objects.query().fetch();
page.toArray().map((row) => row.id);
```

### Prefer async iteration for “iteration triggers evaluation”

If you want iteration itself to run the query:

```ts
const queryset = PostModel.objects.query().filter({ published: true });
for await (const post of queryset) {
    post.title;
}
```
